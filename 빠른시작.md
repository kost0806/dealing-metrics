## 🎯 즉시 시작하기 (3단계)

### 1️⃣ Firebase 설정 (5분)

**Firebase Console (https://console.firebase.google.com/)**
1. "프로젝트 추가" → 프로젝트 이름 입력
2. "Firestore Database" → "데이터베이스 만들기" → **테스트 모드** 선택
3. 프로젝트 설정 → 웹 앱 추가 `</>` → SDK 구성 정보 복사

### 2️⃣ 로컬 설정 (2분)

프로젝트 폴더에서:

```bash
# .env 파일 생성
cp .env.example .env

# .env 파일을 열고 Firebase 정보 붙여넣기
# (위에서 복사한 SDK 구성 정보)
```

### 3️⃣ 실행 (1분)

```bash
# 의존성 설치 (이미 완료됨)
npm install

# 개발 서버 시작
npm run dev
```

브라우저에서 **http://localhost:5173** 접속! 🎉

## 📱 사용 방법

### 첫 화면
1. **사용자 이름** 입력
2. "시작하기" 버튼 클릭

### 딜량 입력
1. **9개의 입력 필드**에 각 회차별 딜량 입력
   - 예시: 100000, 120000, 110000, 130000, 105000, 115000, 125000, 108000, 112000
2. 모두 입력하면 "제출" 버튼 활성화
3. **제출** 클릭

### 통계 확인
자동으로 계산된 통계 확인:
- 📊 **총 딜량**: 9회 합계
- 📈 **회당 평균**: 총합 ÷ 9
- 🎯 **중앙값**: 9개 중 중간값
- 📉 **표준편차**: 딜량 편차
- 🏆 **상위 몇 %**: 다른 사용자와 비교

## 🔧 문제 해결

### Firebase 연결 안됨
```bash
# 1. .env 파일 확인
cat .env

# 2. VITE_ 접두사 확인
# ✅ VITE_FIREBASE_API_KEY=...
# ❌ FIREBASE_API_KEY=...

# 3. 개발 서버 재시작
# Ctrl+C
npm run dev
```

### 데이터 저장 안됨
Firebase Console → Firestore Database → 규칙:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;  // 테스트용
    }
  }
}
```

### 빌드 오류
```bash
# node_modules 재설치
rm -rf node_modules package-lock.json
npm install
```
